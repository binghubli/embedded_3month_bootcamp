# Day8 位操作、宏、位域、大小端、内存对齐笔记
## 1. 基础位运算
& 与：清0、读位
| 或：置1
^ 异或：翻转、无临时变量交换
~ 取反：配合与清零
<< >> 移位：乘除2，无符号/有符号移位补位规则不同

## 2. 寄存器操作宏设计
1. SET_BIT(reg,bit)：置1单bit
2. CLEAR_BIT(reg,bit)：清零单bit
3. READ_BIT(reg,bit)：读取单bit 0/1
4. WRITE_FIELD：多bit域写入，自动屏蔽原有数据
do{}while(0) 消除宏if/else语法坑，参数全部括号包裹避免优先级副作用。

## 3. 位域
struct 成员 : N 占用N比特；
优点：节省RAM；
缺点：存储顺序编译器不统一，网络协议、跨平台禁止使用。

## 4. 大小端
小端(x86/ARM本地)：低地址存低字节
大端(网络协议)：低地址存高字节
转换函数 host_to_be16/32、be16/32_to_host；
用联合体判断本机端序。

## 5. 内存对齐
默认对齐：成员偏移是自身类型整数倍，总大小为最大成员整数倍；
#pragma pack(n) 强制压缩对齐，减小结构体占用；
offsetof 查看成员偏移用于寄存器映射。

## 6. 单元测试覆盖点
1. 单bit置位、清零、读取
2. 多bit域写入覆盖测试
3. 位域存储字节查看
4. 本机端序判断 + 大小端互换
5. 默认对齐与1字节压缩对齐尺寸对比
