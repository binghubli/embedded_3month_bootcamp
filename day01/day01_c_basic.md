# 函数指针（薄弱部分）
## 一、基础核心概念（必背）
### 1. 本质
普通指针：存放**变量内存地址**
函数指针：存放**函数代码段入口地址**
函数加载到内存后，有唯一起始地址，函数名等价于函数入口地址。

### 2. 关键语法拆分（最难，逐段拆解记忆）
标准格式：
```c
返回值类型 (*指针名)(形参类型列表);
```
#### 拆解要点
1. `(*指针名)` 的括号**绝对不能省略**
   - `int *p(int)`：声明一个返回`int*`的普通函数
   - `int (*p)(int)`：声明指向「入参int、返回int」的函数指针
2. 括号内只写形参类型，不用写变量名；写变量名也合法但无意义。

示例1：无参、返回int
```c
int (*pFunc)();
```
示例2：2个int入参、返回int
```c
int (*calc)(int a, int b);
```

## 二、赋值、调用两种写法（背诵）
### 1. 赋值（两种写法完全等价）
```c
int add(int x,int y)
{
    return x+y;
}

int (*p)(int,int);
p = add;     // 写法1：函数名直接赋值
p = &add;    // 写法2：取函数地址，C语言允许，效果一模一样
```

### 2. 通过函数指针调用函数（两种等价写法）
```c
// 写法1：常规解引用调用
int res1 = (*p)(10,20);
// 写法2：直接指针调用（常用）
int res2 = p(10,20);
```
> 记忆：函数指针调用时，`*`可加可不加，编译器自动识别。

## 三、typedef 简化函数指针（嵌入式高频，必背）
原生写法太长，工程全部用`typedef`起别名
### 模板
```c
typedef 返回值 (*类型别名)(形参列表);
```

示例：
```c
// 给该类函数指针起别名 OP
typedef int (*OP)(int,int);

OP p1 = add;   // 直接用别名定义指针，简洁
OP p2;
p2 = sub;
```

## 四、函数指针数组（菜单表、按键回调、状态机标配）
### 1. 定义格式
```c
返回值 (*数组名[数组长度])(形参列表);
```

示例：存放多个运算函数
```c
int add(int a,int b){return a+b;}
int sub(int a,int b){return a-b;}

// 定义2个同类型函数指针构成的数组
int (*op_table[2])(int,int) = {add, sub};
```

### 2. 调用
```c
op_table[0](5,2);  // 执行add(5,2)
op_table[1](5,2);  // 执行sub(5,2)
```

### 3. typedef简化版（推荐背诵）
```c
typedef int (*OP)(int,int);
OP op_table[2] = {add,sub};
```

## 五、回调函数原理（嵌入式必考逻辑）
### 1. 定义
把**函数指针当作函数形参**传入另一个函数，外部不用修改内部代码，就能更换执行逻辑，就是回调。

### 标准模板背诵
```c
// 接收函数指针做参数
void driver(int (*callback)(int))
{
    int ret = callback(100); // 执行外部传入的函数
}

// 实际回调函数
int funcA(int x){ return x*2; }

// 调用，把funcA传入driver
driver(funcA);
```
记忆口诀：**函数指针做形参 = 回调函数**。

## 六、高阶：返回函数指针的函数（难点背诵）
### 原生复杂声明拆解
```c
int (*get_func(int flag))(int,int);
```
逐层拆解：
1. `get_func(int flag)`：这是一个函数，形参int flag
2. 整个函数的返回值：`int (*)(int,int)` 函数指针

### 用typedef简化（必背实操写法）
```c
typedef int (*OP)(int,int);
// 函数返回OP类型（函数指针）
OP get_func(int flag);
```

## 七、易混易错对比表（背诵避坑）
| 声明写法                | 真实含义                                   |
|-------------------------|--------------------------------------------|
| `int *p(int);`          | 普通函数，入参int，返回int*指针            |
| `int (*p)(int);`        | 函数指针，指向入参int、返回int的函数       |
| `int (*p[5])(int);`     | 函数指针数组，存5个同类型函数指针          |
| `int (*(*p)[5])(int);`  | 二级函数指针（极少用）|

## 八、固定背诵口诀（考前速记）
1. 函数名 = 函数入口地址，可以直接赋值给函数指针；
2. `(*p)` 外层括号不能丢，丢了就变成返回指针的普通函数；
3. 赋值：`p=func` / `p=&func` 完全等效；
4. 调用：`(*p)()` / `p()` 两种写法随便用；
5. `typedef` 是函数指针工程化必备，大幅简化代码；
6. 函数指针作形参 = 回调函数，中断、菜单、状态机全靠它；
7. 函数指针数组 = 跳转表，嵌入式菜单一键分发逻辑。

## 九、极简背诵默写模板（可直接抄写）
### 模板1：基础定义+赋值调用
```c
// 1. 目标函数
int func(int a){return a;}
// 2. 定义函数指针
int (*p)(int);
// 3. 赋值
p = func;
// 4. 调用
p(10);
(*p)(10);
```

### 模板2：typedef别名标准模板
```c
typedef int (*FUNC)(int);
FUNC p = func;
```

### 模板3：回调函数固定框架
```c
void test(void (*cb)(void))
{
    cb(); // 回调执行
}
void my_cb(void){}
test(my_cb);
```

### 模板4：函数指针数组（跳转表）
```c
typedef void (*MENU)(void);
MENU menu_tab[] = {func1,func2,func3};
menu_tab[0]();
```

## 十、嵌入式实际用途（记忆用途加深理解）
1. 按键/中断服务函数注册；
2. LCD菜单多级跳转表（函数指针数组）；
3. 分层驱动：上层业务函数不改动底层驱动，用回调注入；
4. 操作系统任务注册、钩子函数。


# 补充static用法
修饰对象	生命周期	作用域	核心用途
局部普通变量	函数调用期（栈）	仅函数内	临时数据，用完即销毁
static 局部变量	整个程序运行期（静态区）	仅函数内	保留历史计数、状态标记、只初始化一次
普通全局变量	程序全程（静态区）	整个工程全部.c	多文件共享数据，易重名
static 全局变量	程序全程（静态区）	仅限当前.c	模块私有变量，隔离封装
普通函数	代码段	全工程可调用	对外接口函数
static 函数	代码段	仅限当前.c 调用	模块内部工具函数，私有化
